Implementation

The examples demonstrate how to use button elements. To use a button style on a link, add the mycal-button class to your link.

To use a different style button on your link, add the special button class in addition to mycal-button:

  • mycal-button-primary-alt
  • mycal-button-secondary
  • mycal-button-gray
  • mycal-button-outline
  • mycal-button-outline-inverse
  • mycal-button-disabled
  • mycal-button-big

For example, a secondary button style would use the following code: <a class="mycal-button mycal-button-secondary" href="/my-link">My button</a>

Accessibility

  • Buttons should display a visible focus state when users tab to them.
  • Avoid using <div> or <img> tags to create buttons. Screen readers don't automatically know either is a usable button.
  • When styling links to look like buttons, remember that screen readers handle links slightly differently than they do buttons. Pressing the Space key triggers a button, but pressing the Enter key triggers a link.

Usability

When to use
  • Use buttons for the most important actions you want users to take on your site, such as "download," "sign up," or "log out."
When to consider something else
  • If you want to lead users between pages of a website. Use links instead.
  • Less popular or less important actions may be visually styled as links.
